# ADR 002 — Why This Job API?

## Status

Adzuna Selected

---

## Context

The project requires live job market data.

The API should provide:

- Job titles
- Company names
- Locations
- Salaries (when available)
- Job descriptions
- Search capability

---

## Evaluation Criteria

- Reliability
- Documentation
- Rate limits
- Free tier
- Data quality
- Geographic coverage
- Ease of integration

---

## Decision based on

Strong analytics capabilities
Historical employment data
Salary-focused endpoints
Better long-term value for DataScope
Well-documented REST API
Supports the project's evolution from ETL → Analytics → Dashboard → Website

## Rationale

The selected API should balance educational value with realistic production usage.

The objective is to simulate an industry ETL pipeline rather than maximize dataset size.
